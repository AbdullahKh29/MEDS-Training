# RISC-V C Extension (Compressed Instruction Set)

## Introduction

The RISC-V C (Compressed) Extension adds 16-bit instruction encodings alongside the standard 32-bit instructions. Its primary goal is to reduce program size while maintaining compatibility with the base ISA. Processors supporting the C extension can execute both 16-bit and 32-bit instructions transparently.

---

## What the C Extension Adds

Normally, RISC-V instructions occupy 32 bits. The C extension introduces compressed 16-bit versions of commonly used instructions, allowing programs to occupy less memory.

This improves:

- Code density
- Instruction cache utilization
- Memory bandwidth efficiency
- Overall system performance, especially in embedded systems

The compressed instructions perform the same operations as their 32-bit equivalents but use shorter encodings whenever possible.

---

## Common Compressed Instructions

Some commonly used compressed instructions include:

- `c.addi` – Compressed version of `addi`
- `c.li` – Load an immediate value into a register
- `c.mv` – Move data between registers
- `c.lw` – Compressed load word
- `c.sw` – Compressed store word
- `c.j` – Unconditional jump
- `c.jr` – Jump to register
- `c.beqz` – Branch if register equals zero
- `c.bnez` – Branch if register is not zero

These instructions are automatically generated by the assembler when appropriate.

---

## Practical Applications

The C extension is especially useful in:

- Embedded systems
- IoT devices
- Microcontrollers
- Battery-powered devices
- Systems with limited flash or RAM

Smaller programs require less memory and often improve cache performance, leading to lower power consumption and faster execution.

---

## Conclusion

The RISC-V C Extension improves code density by introducing 16-bit compressed instructions without changing program behavior. It reduces memory usage, improves cache efficiency, and lowers power consumption, making it one of the most widely implemented RISC-V extensions for embedded and resource-constrained systems.
