# Lecture 6: Hardware Description Languages (Verilog)
* **Concurrency**: Understanding that lines of code in Verilog execute simultaneously, unlike sequential software code.
* **Blocking (=) vs. Non-Blocking (<=) Assignments**: 
    * Use **Non-Blocking** ($<=$) for Sequential logic/registers.
    * Use **Blocking** ($=$) for Combinational logic.
* **Synthesis**: The tool-driven process of mapping behavioral descriptions to actual gate-level netlists.
